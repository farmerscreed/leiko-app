# Screen — Caregiver Home

Sourced from D8 §4.5 (Family Circle / Home) + D6 US-23 / US-32 / US-49.

---

## Audience
Caregiver. Default tab on app launch (Tab Bar position 1).

## Purpose
At-a-glance status of every family member the caregiver is looking after. The single screen the caregiver should be able to glance at and feel reassured.

---

## Layout (top to bottom)

| Element | Spec |
| --- | --- |
| Header | App logo (centred or left-aligned, 32pt) + Settings icon (top-right, Phosphor `GearSix`, 24pt) |
| Anomaly Banner (conditional) | If any parent in the circle has `calm_concerned` or `confirmed_urgent` state: full-bleed banner per `docs/10-anomaly-logic.md` §5 — only ONE banner visible at a time (most-severe wins) |
| Weekly Snapshot card | A single card summarising the week so far for the most-active parent. `type.title` "This week so far"; small inline trend chart (last 7 days); `type.body-m` summary line generated by Tier C (`docs/07-ai-assistant.md`) |
| Reading Cards list | Vertically scrolling list of `ReadingCard` (`docs/03-components/reading-card.md`), one per parent. Cards stack with `spacing.xl` between them. |
| Floating action button (FAB) | Bottom-right, `button.accent`, Phosphor `UserPlus` 20pt + label "Invite family". **Only visible if caregiver is the `family_owner`** of at least one family with capacity remaining (Plus = up to 5 caregivers). |
| Tab bar | Visible — Home tab active |

## Behaviour

- **Pull-to-refresh** triggers `/sync`. Auto-refresh on app foreground. TanStack Query handles cache + retries.
- **Realtime updates**: Supabase Realtime subscription on `public.readings WHERE family_id IN (caregiver's families)`. New readings push into the Reading Card live.
- Card tap → routes to Reading Detail with `reading_id`.
- Banner tap → routes to Reading Detail of the triggering reading (anomaly) OR to a context-specific screen (e.g. silent state → "Send a friendly hello" sheet).

## Free-tier notes

Per `docs/09-paywall-and-iap.md`:
- Free tier shows latest reading + last 7 days. Anything beyond requires Plus.
- Unobtrusive "Get proactive alerts with Leiko Plus" link near the Weekly Snapshot when free-tier user has 3+ readings (introduces paywall without blocking the basic surface).
- **Latest reading is NEVER paywalled.**

## Voice

Per `docs/05-voice-and-claims.md`:
- Reassuring tone by default; calm-concerned only when a parent is in that state.
- Headlines sentence-case. Never countdowns. Never urgency timers.
- Per CLAUDE.md anti-pattern list: **No count badges, "new" dots, or unread indicators on Learn cards** — and the same here for reading cards (no "3 new" badge).

---

## Anti-patterns (CLAUDE.md)

- **Don't use red for normal-state UI.** Red (crimson) is reserved for confirmed-urgent on the reading card stripe + banner only.
- **Don't add count badges.** No "12 new" pill on the family circle.
- **Don't auto-fill medical data fields based on prior values.**
- **Don't add fear-based push notifications.** This screen reflects what the user receives — keep it calm.

---

## Accessibility

- Reading cards: each one is a single tappable region with composed `accessibilityLabel` ("Mum's most recent reading: 128/82 mmHg, 6 above her average. Tuesday 7:42am. Tap for details.").
- FAB: `accessibilityLabel: "Invite a family member"`.
- Anomaly banner: `accessibilityLiveRegion="polite"` so screen-reader announces when state changes.
- VoiceOver order: header → banner (if present) → weekly snapshot → reading cards top-to-bottom → FAB.

---

## Sprint 7 acceptance criteria
- Empty + populated states render with correct tokens.
- Realtime new-reading insertion works (test with simulated /sync push).
- Anomaly banner appears for at least one calm-concerned + one confirmed-urgent fixture.
- FAB visible only for family_owner with capacity remaining.
- Voice gate passes.
- Component + integration tests covering empty, populated, anomaly states.

## D8a status — UNCHANGED

D8a §6 SUPERSEDES D8 §4.5 for the **self-buyer track only**. Caregiver Home (this file) is **UNCHANGED** by D8a per the diff convention. The Family Circle metaphor stays. The Take-a-Reading FAB does NOT belong here (caregivers don't take readings — parents do, on the watch — per D8a §6.5 callout).

In **hybrid mode** (a self-buyer who invited a caregiver), the caregiver sees this Home screen with the self-buyer's data as one card in the Family Circle. Per D8a §1.3 hybrid asymmetry: each user sees the metaphor that fits their role.
