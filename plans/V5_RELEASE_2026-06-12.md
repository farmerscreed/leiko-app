## 4. ⚠️ BEFORE you upload — confirm the upload key

During the device test, the **old vc4** installed from Play had a **different
signature** than `leiko-release.jks` (`adb install -r` was refused with
`INSTALL_FAILED_UPDATE_INCOMPATIBLE`). That is the **expected** signature of
**Play App Signing**: Play re-signs with the app key, so the on-device build
won't match your local **upload** key. It strongly implies `leiko-release.jks`
is the registered **upload key** — but confirm it before uploading:

- Play Console → **com.leiko.care** → **Setup → App integrity → App signing**.
  Your **upload key certificate** SHA-256 there must match this build's signer:
  **`CF:65:9F:68:AA:C6:A8:EA:E0:06:8B:E0:2F:CA:47:83:9F:0E:23:53:72:D9:A5:7F:D3:64:E0:38:55:8A:78:65`**
  (re-derive with `keytool -printcert -jarfile app-release.aab`).
- **If they match** → the AAB will upload cleanly.
- **If they don't** → Play rejects with "signed with the wrong key." Either
  `leiko-release.jks` isn't the current upload key (use the correct one), or
  request an **upload key reset** in Play Console and re-upload.

(`leiko-release.jks` is the irreplaceable upload identity — 1Password
"Leiko Android Release Keystore". Never lose it; never commit it.)

## 5. Upload steps (Google Play Console)

Account: everything is under **`primethebrain@gmail.com`**
(`docs/release/android-release.md`). Reference runbook for closed testing:
`docs/release/closed-test-runbook.md`.

1. Play Console → app **com.leiko.care** → **Testing → Internal testing**
   (start here — fastest, no review wait) → **Create new release**.
2. Upload `app-release.aab` (versionCode **5**). It must be **strictly greater
   than the highest versionCode already on the track** — vc4 is the last
   corporate build and the vc5 bench APK was only sideloaded (never uploaded),
   so vc5 is valid. If Play says "version code already used," bump
   `LEIKO_VERSION_CODE` and rebuild.
3. Add release notes, review, **Save → Review release → roll out to Internal**.
4. Add your testers (internal testers list), install via the opt-in link,
   smoke-test on a fresh device.
5. Promote Internal → **Closed** → **Open/Production** when satisfied.
